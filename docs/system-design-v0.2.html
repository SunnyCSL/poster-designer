🏗️ Phase 0: 背景設定 Current

項目定位

Poster Designer 唔係一個純粹俾人類用得 poster editor，而係一個 俾 AI Agent 同人類協作 嘅 poster creation tool。核心價值在於：

  • 人類可以語音/文字描述需求，Agent 聽得明並且執行
  • 人類俾參考圖，Agent vision analyze 拆解成元素
  • Agent 同人類可以 同時編輯同一張 poster，即時睇到對方改動
  • Agent 可以逐個 element 獨立製作，最後 compose 成完整 poster

💡 設計核心信念

呢個工具係 Agent-native，其次先係 human-friendly。即係所有操作 Agent 都可以用 CLI/API call 完成，而人類可以透過 PWA 即時睇到 Agent 做緊咩、參與修改、俾 feedback。兩者係平等嘅 co-editor，唔係 AI generation + human review。

🎯 Phase 1: 目標設定

Functional Requirements

FR-01
語音→設計規格 — 支援語音輸入或文字描述需求，Agent 解析為結構化設計 spec
FR-02
參考圖→拆解 — Agent 可 vision analyze 參考圖，識別文字區塊、圖片位置、顏色、layout 結構，輸出 element list
FR-03
逐元素製作 — Agent 可以逐個 element（text / image / shape）獨立建立、定位、設定樣式
FR-04
元素匯合 — 將所有獨立 elements compose 成一個完整 poster，處理 layer order、對齊、佈局
FR-05
人機同步 — Agent 同人類可同時編輯同一 shared state，改動即時推送俾對方
FR-06
Agent CLI — Nexi 可以用 terminal command 操作所有功能（add-text, add-image, compose, export）
FR-07
Human PWA — 人類喺 iPhone/Mac 上有 Fabric.js 界面，可以 drag/resize/edit elements
FR-08
Export — 匯出 PNG / PDF（/ 未來 Google Slides）
FR-09
版本管理 — 每次改動有 history，可以 undo/redo，可以睇到邊個改咗
FR-10
Template 系統 — 支援 A3/A4/Square/Mobile 等 preset template sizes

Non-Functional Requirements

NFR-01
同步延遲 < 1s — Agent 或人類做改動後，對方要喺 1 秒內見到
NFR-02
Agent-native — 所有操作必須有 API/CLI 接口，唔可以只有 GUI
NFR-03
Mobile-first — PWA 界面以 iPhone 為 primary target，Desktop secondary
NFR-04
輕量 — Python backend 要 fit 喺 Pi 4 3.7GB RAM 入面行
NFR-05
每個 Stage 獨立可測 — 每個 milestone 推上 GitHub Pages 就有 URL 可以 real device 測試

Success Criteria

2.2 Shared Data Model (state.json)

呢個係整個系統嘅核心——Agent 同 PWA 共用嘅 state。

{
  "version": 2,
  "edited_by": null,       // "nexi" | "sunny" | null
  "last_updated": "ISO8601",

  "meta": {
    "template": "A4",
    "width_mm": 297,
    "height_mm": 210,
    "dpi": 300,
    "width_px": 3508,      // calculated
    "height_px": 2480
  },

  "background": {
    "type": "solid",        // "solid" | "gradient" | "image"
    "color": "#ffffff",
    "gradient": null,
    "image": null
  },

  "elements": [
    {
      "id": "el_001",
      "type": "text",       // "text" | "image" | "shape"
      "content": "Grand Opening",
      "style": {
        "fontFamily": "Noto Sans TC",
        "fontSize": 48,
        "fontWeight": "bold",
        "color": "#000000",
        "textAlign": "center",
        "lineHeight": 1.2,
        "letterSpacing": 0
      },
      "position": {
        "x": 148.5, "y": 50,     // mm coordinates
        "width": 200, "height": 60,
        "rotation": 0
      },
      "layer": 3,
      "locked": false,
      "visible": true,
      "group": null
    }
  ],

  "history": [
    {
      "timestamp": "ISO8601",
      "agent": "nexi",        // "nexi" | "sunny"

}
📐 座標系統

所有 position 用 mm 做單位，唔係 pixels。PWA 顯示時根據 device pixel ratio 縮放，compose export 時根據 DPI 轉 pixel。咁樣 Agent 同 PWA 唔會因為 screen size 唔同而讀錯位置。

2.4 SSE 即時同步機制

關鍵設計——點樣做到人機即時同步：

Nexi runs: poster add-text "Hello" --x 100 --y 50
       │
       ▼
poster CLI → POST /api/element → Server updates state.json
                                                │
                                                ▼
                                     Server pushes SSE event:
                                     {"type": "element_added", "id": "el_002"}
                                                │
                     ┌──────────────────────────┘
                     ▼
              Sunny's PWA receives SSE
              → GET /api/state (fetch latest)
              → Fabric.js canvas re-renders
              → Sunny sees "Nexi added a text element"

Sunny drags a text box on PWA
       │
       ▼
PWA → PATCH /api/element/el_002 { "x": 120, "y": 60 }
       → Server updates state.json
       → Server pushes SSE: {"type": "element_updated", "id": "el_002"}
                                                │
                     ┌──────────────────────────┘
                     ▼
              Nexi doesn't use SSE directly
              → Nexi reads state.json when it needs to know current state
              → Or server can notify via webhook/callback
SSE EventPayloadOccurs When
element_added{"id": "el_003", "type": "text"}Nexi/Sunny add element
element_updated{"id": "el_003", "props": ["x","y"]}Position/style change
element_deleted{"id": "el_003"}Element removed
state_replaced{"version": 3}Bulk update / new design
compose_complete{"url": "..."}Compose finished
agent_editing{"agent": "nexi", "action": "..."}Nexi currently editing

2.5 Agent CLI Spec

呢層係 Nexi 嘅主要接口——所有操作透過 terminal command 完成：

poster new [template]        # 新 design (A4/A3/Square/Mobile...)
poster state                  # 顯示當前 state 摘要
poster add-text "內容" --x --y [--font-size 48] [--bold] [--color "#fff"]
poster add-image <path> --x --y [--width 100] [--height 100]
poster add-shape <rect|circle> --x --y --width --height [--bg "#eee"]

poster export --format png|pdf     # 匯出
poster undo                        # 復原
poster history                     # 睇操作記錄
poster load <file.json>           # 載入 state
poster save <file.json>           # 儲存 state
poster upload <image-path>        # 上傳圖片到 server
poster preview                     # 生成 HTML preview（可唔開 server）
poster help                        # 全部 command

2.7 Vision → Element Decomposition Pipeline

呢個係最關鍵嘅 AI 能力——將一張參考圖拆解成可製作的 element list：

Input: reference.jpg

Step 1 — Vision Analyze (mmx vision / vision_analyze)
  → Detect: text regions, image regions, shapes, color palette
  → Output: structured description

Step 2 — NL → Element List (DeepSeek reasoning)
  → Convert description to structured element array
  → Each element: type, content, approximate position, style
  → Output: JSON element list

Step 3 — Agent Execution
  → Loop through element list → call poster add-* for each
  → Position elements based on reference analysis
  → Set colors, fonts, sizes matching reference

Step 4 — Compose + Present
  → poster compose → show result to Sunny
  → Sunny can tweak any element in PWA
✅ 拆解 vs 生成

關鍵區別：唔係叫 AI「生成一張 poster」，而係 AI「分析參考圖 → 拆成獨立元素 → 逐個製作 → 匯合」。呢個 approach 嘅好處係人類可以喺任何一步介入修改，唔係 black box generation。

2.8 Speech → Design Spec Pipeline

Sunny 語音: 「幫我整張藍色背景嘅 poster，
           左上角有 logo，中間大大隻字寫住 Grand Opening，
           下面有日期同地點」

Step 1 — speech-to-text (edge-tts / whisper)
  → "幫我整張藍色背景嘅 poster，左上角有 logo..."

Step 2 — NL → Structured Spec (DeepSeek reasoning)
  → {
      background: { type: "solid", color: "blue" },
      elements: [
        { type: "image", position: "top-left", description: "logo" },
        { type: "text", content: "Grand Opening", position: "center", style: "large" },
        { type: "text", content: "[date] [location]", position: "bottom-center" }
      ]
    }

Step 3 — Agent Execution
  → poster set-bg --color "#1a3a6b"
  → poster add-text "Grand Opening" --x 148 --y 80 --font-size 48 --bold ...
  → poster add-text "2026.06.01 | Central" --x 148 --y 160 ...

Step 4 — Compose + Present
  → poster compose → Sunny 見到成品

⚠️ Phase 4: 風險管理

風險嚴重性可能性緩解
Pi 4 RAM 不足以同時 run FastAPI + PWA + Hermes Gateway 高 中 FastAPI 用 uvicorn 單 worker + asyncio；圖片唔 save 喺 memory；state.json 用 file-based 唔用 DB
SSE 連接斷開（PWA 休眠 / network change） 中 中 EventSource auto-reconnect；PWA on reconnect 全量 fetch state
Human 同 Agent 同時改同一個 element 中 低 Optimistic locking — version check on PATCH；last-write-wins 但 history 有 record
Vision analysis 唔夠準確 中 中 Stage 5 有 human-in-loop review；Sunny 可以喺 PWA 修正任何 element
GitHub Pages 只 serve static files，唔可以 serve API 低 低 PWA 行喺 GitHub Pages；API server 行喺 Pi (Tailscale)；PWA 經 Tailscale IP 連 API
iPhone Safari 背景 tab 暫停 SSE 中 高 Page Visibility API detect → reconnect；PWA service worker 可 keep alive
Compose Engine 唔支援中文字體 中 低 用 fpdf2 + Noto Sans CJK；已有 pdf-report-generation skills 解決過
